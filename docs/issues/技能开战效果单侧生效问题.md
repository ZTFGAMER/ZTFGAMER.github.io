# 技能开战效果单侧生效问题

## 📋 问题概述

战斗引擎在初始化阶段和战斗过程中，存在大量技能效果只对玩家侧（`player`）生效、对敌方侧（`enemy`）完全无效的问题。这导致两种模式下均出现非预期行为：

- **PVE 模式**：AI 敌人即使被分配了部分技能（通过 `rollEnemySkillIds`），其中属于"开战初始化效果"或"侧面限定触发"的技能完全不生效，敌人实际强度低于设计预期。
- **PVP 模式**：双方各自在本地独立模拟同一场战斗，玩家侧技能正常生效，而对手的对应技能（位于 `enemy` 侧）不生效，导致两端模拟结果分叉，出现双方均显示自己胜利的情况。

---

## 🎯 核心问题

### 开战初始化效果缺失 enemy 侧处理

**问题说明**：

- `applyPickedSkillBattleStartEffects` 方法（`CombatEngine.ts`）在战斗开始时统一执行技能的初始化加成
- 方法内部全程使用 `hasPlayerSkill` 检查技能归属，只修改玩家侧物品和英雄属性
- 没有对 enemy 侧执行任何等价的初始化逻辑
- `enemySkillIds` 中即使存在对应技能 ID，其开战加成效果也会被完全跳过

**实现说明**：

- 在 `CombatEngine.ts` 中的 `applyPickedSkillBattleStartEffects` 方法实现

**受影响的技能（开战初始化类）**：

| 技能 ID | 效果描述 | 影响范围 |
|---------|---------|---------|
| `skill1` / `skill2` | 首个/末个护盾物品护盾值 +25 | 护盾类阵容 |
| `skill3` | 所有护盾物品护盾值 +10 | 护盾类阵容 |
| `skill7` | 英雄最大 HP ×1.1 | 所有阵容 |
| `skill49` | 上阵区未满时英雄最大 HP ×2 | 特殊布阵策略 |
| `skill36` | 开战获得 30% 最大 HP 的护盾 | 所有阵容 |
| `skill8` | 首个弹药物品弹药上限 +2 | 弹药类阵容 |
| `skill51` | 所有弹药物品弹药上限 +2 | 弹药类阵容 |
| `skill44` | 单弹药物品时弹药上限 +10 | 单弹药阵容 |
| `skill10` | 所有弹药物品伤害 +10 | 弹药类阵容 |
| `skill16` | 所有武器物品伤害 +8 | 武器类阵容 |

### 战斗过程中触发效果缺失 enemy 侧处理

**问题说明**：

- 战斗循环中多处技能触发逻辑使用 `item.side === 'player' && hasPlayerSkill(...)` 双重判断
- 导致这些技能在物品属于 `enemy` 侧时无法触发，即使 `enemySkillIds` 中存在对应技能
- 部分技能会直接影响战斗走向（如 `skill47` 濒死复活），效果缺失会导致 PVP 双端模拟分叉

**实现说明**：

- 在 `CombatEngine.ts` 中的 `applyAmmoConsumptionSkillTriggers`（弹药消耗触发）和 `applyOnItemUseTriggers`（物品使用触发）等相关方法实现

**受影响的技能（战斗过程类）**：

| 技能 ID | 触发时机 | 效果描述 |
|---------|---------|---------|
| `skill9` | 首次弹药耗尽 | 补充 3 发弹药并触发成长 |
| `skill52` | 弹药耗尽 | 50% 概率复满全部弹药 |
| `skill54` | 每次弹药消耗 | 所有同侧物品伤害 +5 |
| `skill85` | 弹药耗尽 | 相邻物品伤害 +50 |
| `skill47` | 英雄 HP 降至 0 | 濒死复活，恢复 50% 最大 HP |
| `skill4` | 护盾物品触发 | 相邻物品伤害 +4 |
| `skill5` | 护盾物品触发 | 护盾物品护盾值自增 +7 |
| `skill11` | 弹药物品触发 | 相邻物品伤害加成 |
| `skill86` | 武器物品触发 | 对相邻弹药物品充能 |
| `skill87` | 最左武器物品触发 | 对最右物品充能 |
| `skill88` | 开战 5 秒内物品触发 | 额外多重施放 +1 |

## 🔄 系统影响

### PVE 战斗

- 方法: `makeEnemyRunners` 在 `CombatEngine.ts`
- 功能: 生成 AI 敌人物品，配合 `rollEnemySkillIds` 为敌人分配技能
- 数据依赖: 分配的敌人技能存入 `enemySkillIds`，但开战初始化和侧面限定触发均不执行，敌人实际强度低于设计值

### PVP 战斗

- 方法: `applyOpponentSnapshot` 在 `PvpContext.ts`
- 功能: 将对手快照合并为本地战斗快照后进入战斗场景
- 数据依赖: 对手物品作为 `enemy` 侧参与模拟，但对手的技能（即使正确传输）因侧面限定判断而不生效，导致本地模拟与对手本地模拟结果不一致

---

## 📊 影响分析

```
战斗开始
    ↓
applyPickedSkillBattleStartEffects()
    ├─ hasPlayerSkill('skill7') → player 英雄 HP ×1.1  ✅
    └─ hasEnemySkill('skill7')  → 无对应逻辑            ❌
    ↓
战斗循环（每 tick）
    ↓
物品触发
    ├─ item.side === 'player' && hasPlayerSkill('skill9') → 触发  ✅
    └─ item.side === 'enemy'  && hasEnemySkill('skill9')  → 跳过  ❌
```

| 影响维度 | PVE | PVP |
|---------|-----|-----|
| 技能效果完整性 | 敌人技能部分无效，强度偏低 | 对手技能无效 |
| 战斗结果一致性 | 不涉及 | 双端模拟分叉，胜负不一致 |
| 严重程度 | 平衡性问题 | 功能性 bug |

---

## 🎯 修复方向

**开战初始化**：

- 将 `applyPickedSkillBattleStartEffects` 改为支持 `side` 参数，分别对 `player` 和 `enemy` 侧执行
- 对于依赖玩家上下文（`playerBackpackItemCount`、`playerGold`、`playerTrophyWins`）的技能（`skill35`、`skill46`、`skill95`），需在 PVP 快照中增加对手对应字段并传入引擎

**战斗过程触发**：

- 将 `item.side === 'player' && hasPlayerSkill(id)` 模式统一替换为 `hasSkill(item.side, id)` 模式，复用引擎已有的双侧技能判断基础设施
- 为 `skill47`、`skill9`、`skill86` 等有独立状态变量（如 `skill47ReviveTriggered`）的技能，增加对应的 enemy 侧状态变量

---

## 📝 总结

`applyPickedSkillBattleStartEffects` 及战斗循环中大量 `hasPlayerSkill` 单侧判断，导致敌方侧技能在 PVE 中强度偏低、在 PVP 中造成双端模拟分叉，属于同一根因引发的跨模式问题，修复时应统一将相关技能判断改为支持双侧的 `hasSkill(side, id)` 模式，并配套补全 enemy 侧的初始化逻辑和状态变量。

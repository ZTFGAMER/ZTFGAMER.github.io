# 大巴扎 Demo 设计文档（当前实现汇总）

更新时间：2026-03-09  
版本定位：可试玩 Demo（PVE 主流程 + PVP 联机模式）

## 1. 产品定位与体验目标

- 类型定位：背包构筑 + 自动战斗（Shop-battler），以“构筑决策”驱动战斗结果。
- 核心体验：玩家在商店阶段管理金币、布局、买卖与联动，在战斗阶段验证构筑强弱。
- Demo 目标：
  - 跑通 Day1~Day20 的完整 PVE 循环；
  - 跑通独立 PVP 9 天胜场制流程（2~4 人，AI 补位）；
  - 通过技能、事件、特殊商店和新物品建立中后期爆发感与变盘点。

## 2. 当前玩法结构

### 2.1 PVE 主流程（20 天）

1. 进入 Day N 商店。
2. 发放当日金币（按曲线）。
3. 执行当日节点（技能三选一 / 事件二选一 / 特殊商店）。
4. 常规商店购买、刷新、出售、整理与布局。
5. 进入自动战斗。
6. 战斗结算：胜利推进天数，失败扣心；心为 0 则重开。

### 2.2 PVP 流程（独立模式）

- 入口：`MenuScene` 选择联机对战。
- 大厅：创建/加入房间，2~4 人，不足由 AI 补位。
- 每天流程：90 秒商店构筑 -> 上传快照 -> 配对对手 -> 自动战斗。
- 赛制：固定循环对战，共 9 天，按胜场排名。
- 结算：`PvpResultScene` 展示名次，支持“再来一局”或返回主菜单。

## 4. 内容系统设计

### 4.1 物品池（Vanessa compact，24 件）

配置来源：`data/vanessa_items_compact.json`

- 青铜起始（6 件）：短剑、圆盾、弹药袋、木弓、匕首、连发镖。
- 白银成长（9 件）：长盾、手弩、长剑、弹药堆、回旋镖、大手里剑、头盔、弹弓、飞刃。
- 黄金核心（9 件）：皇家佩剑、尖刺金盾、超级手雷、超级弩机、黄金袖箭、切割镰刀、英雄铠甲、攻城重炮、暗影斗篷。

新增 6 件代表物品（已接入战斗逻辑）：
- `头盔(item19)`：受击充能（1 秒）。
- `弹弓(item20)`：弹药耗尽时 5 倍伤害。
- `飞刃(item21)`：相邻物品使用时立即额外使用。
- `英雄铠甲(item22)`：受击立即使用。
- `攻城重炮(item23)`：补弹叠加伤害。
- `暗影斗篷(item24)`：强化右侧攻击物品连发。

### 4.2 技能三选一系统

- 每日计划字段：`skill_system.dailyDraftPlan`。
- 规则：20 天内按天切换“技能/事件/特殊商店”节点，技能节点可按 Bronze/Silver/Gold 概率抽取。
- 关键节奏：
  - Day1 强制技能，且限定初始职业倾向；
  - Day4/7/10/13/16/19 为关键技能补强日；
  - Day12 后技能池基本进入 Gold 主导。

### 4.3 事件系统

- 事件池：`event1~event36`（当前配置含跨天与条件事件）。
- 近期新增高影响事件：
  - `event25` 投资金币，次日双倍返还；
  - `event26` 上阵区按双倍出售（按出售价口径结算）；
  - `event27` 当天所有合成改为随机；
  - `event28` 3 天后获得 `(day+3)*5` 金币；
  - `event29` 5 天后上阵区全体升级；
  - `event34` 预支未来 2 天基础收入的 60%；
  - `event35` 转化等级最高物品；
  - `event36` 升级等级最低物品。

## 5. 关卡与对手设计

- 敌人来源：`combat_runtime.enemyTeachingByDay`，Day1~Day20 每天模板化阵容。
- 设计目标：按天展示不同压制点（护盾、弹药、连发、斩杀、爆发）。
- 后期难度：通过高等级模板 + 敌方平均品质曲线提高稳定性，而非纯血量膨胀。

## 6. 交互与可读性规范

- 设计基准：`640x1384`，单格 `128x128`，CSS 统一缩放适配。
- 文本字号统一来自 `text_sizes`，可在 debug 页调参。
- 战斗统计已支持“触发次数”口径，按真实触发事件累计。
- 技能/事件界面支持：
  - 显示当前金币；
  - 单次刷新（刷新后按钮消失）；
  - 强行离开二次确认。

## 7. 技术实现边界

- 引擎：PixiJS，`preference: 'webgpu'` 优先，自动回退 WebGL。
- 架构：数据驱动（JSON）+ 场景状态机 + 战斗逻辑与渲染分离。
- 存档：PVE 与 PVP 隔离；PVP 开始前备份并在结束后恢复 PVE 存档。
- 图标路径约定：`resource/itemicon/vanessa/{item.id}.webp`（工程内图标命名已统一为 `item*`）。

## 8. 当前 Demo 结论

- PVE：已形成“商店构筑 -> 战斗验证 -> 数值反馈 -> 下一天”的稳定闭环。
- PVP：已完成可玩闭环（匹配、快照、9 天循环、排名、回流菜单）。
- 系统层：经济、技能、事件、特殊商店、新物品联动已联通，具备 Demo 演示完整性。

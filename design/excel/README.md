# 配置表规范文档

本目录存放大巴扎项目的所有策划配置表（Excel），供策划、程序、设计同学共同维护。

---

## 文件列表

| 文件名 | 说明 |
|---|---|
| `WeaponCard.xlsx` | 武器卡（物品）配置 |
| `SkillCard.xlsx` | 战斗技能卡配置（含效果参数） |
| `SkillDraft.xlsx` | 技能抽选系统配置（每日计划 + 技能池） |
| `HeroClass.xlsx` | 职业配置 |
| `ShopConfig.xlsx` | 商店配置（每日进度 / 物品定价 / 随机规则） |
| `GameParam.xlsx` | 全局游戏参数（战斗运行时 / PVP规则 / 通用数值） |
| `EnemyByDay.xlsx` | 冒险模式敌人按天配置（物品池 / 每日卡数与模板） |

## 表格结构规范

### 1. 固定行布局

每张 Sheet 的前 4 行固定如下，**数据从第 5 行开始**：

| 行号 | 用途 | 说明 |
|---|---|---|
| 第 1 行 | **表名** | 合并单元格，深色背景，标注当前 Sheet 所属的配置表名称 |
| 第 2 行 | **英文字段名** | 程序读取使用的字段标识符，深色背景白字 |
| 第 3 行 | **字段类型** | 标注每列的数据类型，灰色斜体 |
| 第 4 行 | **中文字段名** | 供策划阅读的中文说明，浅灰背景 |

### 2. 固定列布局

每张 Sheet 的前两列固定如下：

| 列号 | 字段名 | 类型 | 规则 |
|---|---|---|---|
| 第 1 列 | `#` | 注释列 | **永远为空**，不填任何数据，仅作视觉分隔 |
| 第 2 列 | `Id` | `int` | **自增整数**，从 1 开始，每行 +1，不可重复、不可跳号 |

### 3. 字段命名规范

- 使用**大驼峰命名法**（UpperCamelCase），例如：`NameCn`、`StartingTier`、`CooldownMs`
- **禁止**使用特殊符号，包括但不限于：`( ) % / \ [ ] { } @ # * + - .`
- 单位信息用后缀表达，不用括号：
  - 毫秒 → `XxxMs`（例：`CooldownMs`）
  - 百分比 → `XxxPct`（例：`TierBronzePct`）
- 中英文对应列命名：中文字段加 `Cn` 后缀，英文字段加 `En` 后缀，例如：`NameCn`、`NameEn`

### 4. 数据类型约定

| 类型标注 | 说明 | 示例 |
|---|---|---|
| `int` | 整数 | `1`, `2500`, `-1` |
| `float` | 浮点数 | `0.5`, `1.1` |
| `bool` | 布尔值，必须大写 | `TRUE` 或 `FALSE` |
| `string` | 字符串 | `"Bronze"`, `"warrior"` |
| `list[int]` | 整数数组，用 ` \| ` 分隔 | `6 \| 15 \| 24` |

### 5. 颜色编码约定

品质颜色：

| 品质 | 背景色 |
|---|---|
| Bronze（青铜） | `#CD7F32` |
| Silver（白银） | `#A8A8A8` |
| Gold（黄金） | `#DAA520` |
| Diamond（钻石） | `#4DB8DB` |

职业颜色：

| 职业 | 背景色 |
|---|---|
| warrior（战士） | `#4472C4` |
| archer（弓手） | `#70AD47` |
| assassin（刺客） | `#C00000` |
| utility（辅助） | `#7030A0` |
| 中立 | `#888888` |

## 修改规范

1. **新增行**：在末尾追加，`Id` 列继续自增，不允许在中间插入或复用已删除的 Id
2. **删除行**：直接删除整行，不留空行，不允许将 Id 置空保留占位
3. **新增字段**：在末尾追加列，同步补充第 2~4 行的英文名、类型、中文名
4. **修改字段名**：英文字段名（第 2 行）一旦交付程序对接后，修改前须与程序同步确认
5. **数值修改**：直接修改对应单元格，无需额外标注（版本由 Git 管理）

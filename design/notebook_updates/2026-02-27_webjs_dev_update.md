# BigBazzar 项目更新（2026-02-27）

## 今日开发摘要

1. **1D 网格重构完成（P0/P1）**
   - 网格体系切换为单行语义：战斗区与背包均为 `1x6`。
   - 物品尺寸规范统一为 `1x1 / 2x1 / 3x1`。
   - `GridSystem / VirtualGrid / AutoPack / GridZone / DragController / ShopScene / BattleScene` 已迁移并适配。

2. **商店布局约束与居中修复**
   - 战斗区/背包区按可见列宽动态居中，修复旧列数偏移问题。
   - 商店改为“紧贴居中”排布，并增加总宽约束 `<= 6`，避免同屏出现两个大型占宽组合导致溢出。
   - 商店图标高度、描边策略与内容边界框已与 1D 新格子比例对齐。

3. **合成与挤出规则修复（含来源 footprint 回填）**
   - 合成规则收口为“命中目标高亮、落手触发”，并优先于挤出逻辑。
   - 修复单行挤出方向异常，新增“优先回填拖拽来源 footprint”策略，解决中型横向拖拽时空位回填错误。
   - 背包/战斗区跨区拖拽、挤出与高亮行为已按 1D 规则统一。

4. **战斗效果通道拆分（进行中）**
   - 效果通道拆为：
     - **主角通道**：伤害、护盾、治疗、灼烧、中毒、再生
     - **卡牌通道**：冻结、减速、加速（作用于物品充能层）
   - `CombatEngine` 已接入卡牌状态持续时间与充能修正；`BattleScene` 支持效果目标投射到英雄或物品。

5. **验证状态**
   - `npm test`：通过
   - `npm run build`：通过
